The sample demonstrates the use of the navigation protocol and the ms-drive-to and ms-walk-to Uri schemes. You can use the navigation protocol to write an app that requests driving or walking directions, or to write an app that handles requests for driving or walking directions.

The solution contains 2 projects:

  • GetDirectionsWP8 demonstrates how to request directions by launching a Uri that uses the ms-drive-to or ms-walk-to Uri scheme. The sample assembles the Uri from its elements and then displays the complete Uri that is launched.

  • GiveDirectionsWP8 demonstrates how to write an app that handles requests for directions that use the ms-drive-to or ms-walk-to Uri scheme. The sample uses a Uri mapping class to redirect the request to a specific page of the app. Then it displays the values that were passed in the Uri. (However the sample does not demonstrate how to provide turn-by-turn directions.)

For more info, see the following topics:

  • How to request or provide directions for Windows Phone 8

  • Auto-launching apps using file and URI associations for Windows Phone 8

Build the sample

  1. Start Visual Studio and select File > Open > Project/Solution.

  2. Browse to the directory in which you unzipped the sample. Double-click the solution (.sln) file.

    The solution opens in Visual Studio.

  3. Use Build > Rebuild Solution to build the sample.

Run the sample

  • To run the app with debugging, press F5 or use Debug > Start Debugging.

    - or -

  • To run the app without debugging, press Ctrl+F5 or use Debug > Start Without Debugging.
